DiskPart CLI Reference
======================

USAGE
  DiskPart                        Open GUI (device selector window)
  DiskPart <command> [options]    Run CLI command, no GUI opened

The same binary is used for both modes.  No arguments = GUI.
Arguments = CLI (only dos.library is opened; Intuition/GadTools not needed).

Output can be redirected:  DiskPart LISTDEV >ram:devs.txt

RETURN CODES
  0   RETURN_OK    Command completed successfully
  5   RETURN_WARN  Usage error (bad or missing argument combination)
 10   RETURN_ERROR Runtime error (device open failed, ReadArgs error, etc.)
 20   RETURN_FAIL  Fatal error (dos.library could not be opened)


COMMANDS
--------

LISTDEV [UNITS]
  List all block device drivers found on this system.

  No options: instant scan — walks AmigaDOS DosList and exec DeviceList,
  no I/O performed.  Shows driver name and version if loaded.

  UNITS: also probe each driver for responding units.  This opens each
  device in turn (slow — can take several seconds per driver).  Shows
  the same unit info as the GUI's unit-selection window:
  unit number, vendor/product string, and disk size.

  Examples:
    DiskPart LISTDEV
    DiskPart LISTDEV UNITS

  Sample output (LISTDEV):
    Block devices: 3

    uae.device               v37.0
    scsi.device              v37.15
    ide.device

  Sample output (LISTDEV UNITS):
    Block devices: 3

    uae.device               v37.0
      Probing units for uae.device...
      Unit 0   WDC WD5000AAKS   500 GB (476 MB)

    scsi.device              v37.15
      Probing units for scsi.device...
      (no units found)

    ide.device
      Probing units for ide.device...
      Unit 0   SAMSUNG SV1604N   160 GB (152 MB)


DEV <device>:<unit>
  Specify the target device and unit for a command.
  Must be combined with a command (e.g. INIT).
  Format: device name followed by colon and unit number.

  Example:  DiskPart DEV uaehf.device:3 INIT NEW

IMAGE <path>
  Use an image file as the target instead of an exec.device unit.
  The file is treated as a 512-byte-block disk; geometry is derived
  as 16 heads x 63 sectors with cylinders = filesize / (512*16*63).
  Most commands that accept DEV also accept IMAGE.

  Image files are limited to ~2 GB by AmigaOS dos.library Seek().

  Example:  DiskPart IMAGE test.hdf INFO
            DiskPart IMAGE test.hdf INIT NEW

CREATE SIZE=<n>[K|M|G]
  Create a new image file of the given size, then open it.
  Must be combined with IMAGE=<path>.  Uses SetFileSize() — sparse on
  filesystems that support it (SFS, PFS), pre-allocated on FFS.

  CREATE may stand alone (just create the file) or be combined with
  INIT NEW etc. on the same line to also write a fresh RDB.

  Example:  DiskPart IMAGE fresh.hdf CREATE SIZE=100M
            DiskPart IMAGE fresh.hdf CREATE SIZE=100M INIT NEW

IMAGEOUT <path>
  Dump the open target (DEV or IMAGE) to a new image file at <path>.
  Every block of the target from block 0 onward is copied byte-for-byte.
  Existing file at <path> is overwritten.

  Output is purely sequential (no Seek), so files larger than 2 GB are
  fine PROVIDED the destination filesystem supports them (SFS, PFS3,
  FFS-NSD, FFS post-OS3.5).  Older FFS will fail with a write error
  near the 2 GB mark.  A warning is printed up front for sources >2 GB.

  Progress prints every 5% (or every 50 MB when total is unknown).
  Press Ctrl+C to cancel; the partial output file is left on disk.

  Example:  DiskPart DEV uaehf.device:0 IMAGEOUT=ram:disk.hdf
            DiskPart IMAGE fresh.hdf IMAGEOUT=ram:fresh.copy

IMAGEIN <path>
  Restore an image file at <path> back onto the open target.
  The image is read sequentially until EOF (no Seek), so files larger
  than 2 GB are fine if the SOURCE filesystem supports them.
  Each Read result must be a multiple of the device block size, and
  total blocks must fit within the target.
  DESTRUCTIVE: prompts for Y/N confirmation unless FORCE is given.
  Press Ctrl+C to cancel; the destination is left in a partial state.

  Example:  DiskPart DEV uaehf.device:0 IMAGEIN=ram:disk.hdf FORCE

MOUNTLIST <path>
  Write an AmigaDOS MountList text file at <path> describing every
  partition in the open target's RDB (one entry per partition, with the
  full DosEnvVec: Surfaces, BlocksPerTrack, LowCyl/HighCyl, DosType,
  Mask, MaxTransfer, BootPri, ...).  Read-only - the disk is not changed.
  Where the RDB carries a matching filesystem handler path, a FileSystem
  line is added so C:Mount can load it.  Existing file is overwritten.

  Example:  DiskPart DEV scsi.device:0 MOUNTLIST=ram:MountList

INIT NEW
  Create a fresh RDB on the specified device, auto-detecting geometry
  from TD_GETGEOMETRY.  Writes immediately (no separate WRITE step
  needed for this command).

  If an existing RDB is found, a Y/N question is shown before
  overwriting.  FORCE auto-answers Y to all questions.

  Example:  DiskPart DEV uaehf.device:3 INIT NEW
            DiskPart DEV uaehf.device:3 INIT NEW FORCE

INIT NEWGEO
  Update the RDB cylinder count to match the actual disk size.
  Use after replacing a disk with a larger one (clone + expand).
  Keeps existing heads/sectors from the RDB; only the cylinder
  count and hi_cyl are updated.  If the disk is the same size or
  smaller, reports "no update needed" and exits cleanly.

  A Y/N question is shown before writing.  FORCE auto-answers Y.

  Example:  DiskPart DEV uaehf.device:3 INIT NEWGEO
            DiskPart DEV uaehf.device:3 INIT NEWGEO FORCE

FORCE
  Global switch.  Auto-answers Y to all Y/N safety questions.
  Useful for scripting or when you are certain of the operation.

INFO
  Read and display the RDB, partition table, and filesystem drivers.
  Read-only — does not modify the disk.

  Example:  DiskPart DEV uaehf.device:3 INFO

SMART
  Read ATA SMART attribute data from the drive via SCSI ATA
  PASS-THROUGH (12).  Displays all active attributes:

    ID  Attribute                 Cur Wst Raw
    --- ------------------------- --- --- ----------
      1 Read Error Rate           100 100          0
      9 Power-On Hours            100  97       1234
    194 Temperature (C)            35   0         35
    197 Current Pending Sectors   200 200          0

  Cur = current normalised value (1-253, higher is usually better).
  Wst = worst recorded value.
  Raw = vendor raw counter (interpretation varies by attribute).
  PRE-FAIL tag is shown when the attribute is flagged as pre-failure
  type (as opposed to advisory).

  Requires an ATA drive accessible via HD_SCSICMD.  USB and
  pure-SCSI targets typically will not respond.

  Example:  DiskPart DEV uaehf.device:0 SMART
            DiskPart DEV uaehf.device:0 SMART >ram:smart.txt

BACKUP <file>
  Read the RDSK block and save it as a raw 512-byte backup file.
  Safe — read-only operation.

  Example:  DiskPart DEV uaehf.device:3 BACKUP ram:rdb.bak

RESTORE <file>
  Restore a single-block backup to block 0.  Two Y/N questions
  are asked before writing (FORCE auto-answers both).

  Example:  DiskPart DEV uaehf.device:3 RESTORE ram:rdb.bak
            DiskPart DEV uaehf.device:3 RESTORE ram:rdb.bak FORCE

BACKUPEXT <file>
  Save all RDB blocks (RDSK through rdb_HighRDSKBlock) to an
  extended backup file (ERDB format — same as the GUI).
  Includes all PART, FSHD and LSEG blocks.  Read-only.

  Example:  DiskPart DEV uaehf.device:3 BACKUPEXT ram:rdb_ext.bak

RESTOREEXT <file>
  Restore an extended backup (ERDB format).  Validates magic,
  version and block-size before writing.  Two Y/N questions
  (FORCE auto-answers both).

  Example:  DiskPart DEV uaehf.device:3 RESTOREEXT ram:rdb_ext.bak
            DiskPart DEV uaehf.device:3 RESTOREEXT ram:rdb_ext.bak FORCE

ADDPART NAME=<n> LOW=<cyl|START|NEXT|size> HIGH=<cyl|END|+size>
         [TYPE=<dostype>] [BOOTPRI=<n>] [BOOTABLE] [ENFORCESIZE]
         [BLOCKSIZE=<n>]
  Add a partition to the existing RDB, then ask Y/N before writing.
  FORCE skips the write question.  See SCRIPT COMMANDS for full
  description of LOW/HIGH/TYPE/BLOCKSIZE syntax.
  If HIGH would run into an existing partition, it is clamped to fill
  the gap up to that partition.  ENFORCESIZE disables clamping: the
  requested size must fit or the command fails.

  Examples:
    DiskPart DEV uaehf.device:3 ADDPART NAME=DH0 LOW=START HIGH=+100M TYPE=PFS3 BOOTABLE
    DiskPart DEV uaehf.device:3 ADDPART NAME=DH1 LOW=START HIGH=END   TYPE=PFS3 FORCE

ADDFS TYPE=<dostype> [FILE=<path>] [VERSION=<hex>] [STACKSIZE=<n>]
  Add a filesystem driver entry to the existing RDB, then ask Y/N
  before writing.  FORCE skips the write question.

  Examples:
    DiskPart DEV uaehf.device:3 ADDFS TYPE=PFS3 FILE=L:pfs3aio
    DiskPart DEV uaehf.device:3 ADDFS TYPE=PFS3 FILE=L:pfs3aio FORCE

GROW <drive> <size[KMG]|END>
  Extend a partition and grow its filesystem (FFS/SFS/PFS) to match, in
  one step, then ask Y/N before committing.  FORCE skips the question.
  <size> is the amount to ADD (e.g. 100M); if it exceeds the free space
  after the partition it is clamped to the maximum.  END (or MAX) uses all
  the free space.  The RDB is written immediately; FFS remounts live, SFS/PFS
  need a reboot.  See SCRIPT COMMANDS for the full description.

  NOUNMOUNT grows the volume in place instead of unmounting it first.  Use
  this for the boot partition (or any volume with open files), which cannot
  be unmounted.  The grow still runs with the handler inhibited, but the
  volume is then left locked and a REBOOT is mandatory before it is usable -
  do not access it before rebooting.

  Examples:
    DiskPart DEV uaehf.device:3 GROW DH1 100M
    DiskPart DEV uaehf.device:3 GROW DH1 END FORCE
    DiskPart DEV scsi.device:0 GROW DH0 END NOUNMOUNT  ; boot partition

ZEROPART NAME=<drive>
  Overwrite every block of the named partition with zeros.  Does not change
  the RDB or partition table — only the partition's data blocks are written.
  Useful before imaging a disk: zeroed blocks compress far better than
  stale filesystem data, so the resulting image file will be much smaller.

  FORCE skips the Y/N confirmation prompt.
  No WRITE is needed; changes take effect immediately.

  Examples:
    DiskPart DEV scsi.device:0 ZEROPART NAME=DH1
    DiskPart DEV scsi.device:0 ZEROPART NAME=DH1 FORCE

SCRIPT <file> [DRYRUN] [FORCE]
  Read and execute commands from a text file, one per line.
  Comments start with # or ;.  Blank lines are ignored.
  Commands accumulate changes in memory; only WRITE commits to disk.

  DRYRUN: parse and validate, print what would happen, no writes.
  FORCE:  suppress all warnings (same as global FORCE flag).

  Example script (ram:setup.txt):
    # Initialise uaehf.device unit 0
    OPEN uaehf.device 0
    INIT NEW
    ADDPART NAME=DH0 LOW=2 HIGH=511 TYPE=DOS3 BOOTPRI=0
    ADDPART NAME=DH1 LOW=512 HIGH=1023 TYPE=PDS3
    ADDFS TYPE=DOS3 FILE=L:pfs3aio
    WRITE

  Run with:  DiskPart SCRIPT ram:setup.txt
         or: DiskPart SCRIPT ram:setup.txt DRYRUN
         or: DiskPart SCRIPT ram:setup.txt FORCE

  See EXAMPLES section at the bottom of this file for a complete
  disk-preparation script using PFS3.


SCRIPT COMMANDS
---------------
These commands are only valid inside a script file.

OPEN <device> <unit>
  Open a block device for subsequent commands.
  Example:  OPEN uaehf.device 3

OPEN FILE <path>
  Open an image file as the block device for subsequent commands.
  The file is treated as a 512-byte-block disk (16 heads x 63 sectors).
  Image files are limited to ~2 GB by AmigaOS Seek().
  Example:  OPEN FILE work:images/test.hdf

CREATE FILE <path> SIZE=<n>[K|M|G]
  Create a new image file of the given size and open it.
  Existing file with the same name is overwritten.
  Subsequent commands (INIT NEW, ADDPART, ...) operate on the new file.
  Example:  CREATE FILE work:images/fresh.hdf SIZE=100M

IMAGEOUT FILE=<path>
  Dump the currently-open target to a new image file at <path>.
  Block-for-block copy from block 0 to end of target.
  Sequential write — files >2 GB work if destination filesystem supports
  it (SFS, PFS3, FFS-NSD, FFS post-OS3.5).
  Honours DRYRUN (no file is created).
  Example:  IMAGEOUT FILE=ram:disk.hdf

IMAGEIN FILE=<path>
  Restore an image file at <path> back onto the currently-open target.
  Streamed read until EOF — files >2 GB work if the source filesystem
  supports it.  Each chunk must be block-aligned; total must fit.
  DESTRUCTIVE: prompts Y/N (auto-yes if script run with FORCE).
  Re-reads the RDB after the restore so subsequent INFO is accurate.
  Honours DRYRUN.
  Example:  IMAGEIN FILE=ram:disk.hdf

MOUNTLIST FILE=<path>
  Write an AmigaDOS MountList text file describing every partition in the
  currently-open RDB (full DosEnvVec per partition; FileSystem line added
  when the RDB carries a matching handler path).  Read-only.
  Honours DRYRUN (no file is created).
  Example:  MOUNTLIST FILE=ram:MountList

INFO
  Read and display the RDB and partition table.  Read-only.

INIT NEW
  Create a fresh RDB on the opened device, auto-detecting geometry.
  Overwrites any existing RDB (warns unless FORCE is set).

INIT NEWGEO
  Update the RDB cylinder count to match the actual disk size.
  Keeps existing heads/sectors; only cylinder count and hi_cyl change.
  No-op if disk is the same size or smaller.

ADDPART NAME=<n> LOW=<cyl|START|NEXT|size> HIGH=<cyl|END|+size>
         [TYPE=<dostype>] [BOOTPRI=<n>] [BOOTABLE] [ENFORCESIZE]
         [BLOCKSIZE=<n>]
  Add a partition entry to the in-memory RDB.
  TYPE defaults to DOS3 (0x444F5303) if not specified.
  TYPE can be a hex value (0x444F5303 or $444F5303) or a short
  DOS-type string: DOS3, PDS3, SFS0, PFS3, etc. (3 chars + digit).

  LOW=START   First available cylinder from the beginning of the usable
              area (lo_cyl), skipping any partitions already there.
              Safe to use for every partition — each one finds the next gap.
  LOW=NEXT    Start immediately after the last defined partition
              (highest hi_cyl + 1).  Same as START for contiguous layouts.
  LOW=300M    Start at the cylinder 300 MB into the disk.  A K/M/G suffix
              means a byte offset; a bare number is still a cylinder number.

  HIGH=END    Fill to the last usable cylinder on the disk.
  HIGH=+100M  Set HIGH so the partition is exactly 100 MB.
              Suffix: K = kilobytes, M = megabytes, G = gigabytes.
              The size is rounded DOWN to a whole number of cylinders.

  If the chosen HIGH (including END or a +size) would run into an
  existing partition, HIGH is clamped to one cylinder before it, so the
  new partition fills the gap exactly.  A warning reports the new HIGH.
  ENFORCESIZE  Disable clamping: if the requested span overlaps an
               existing partition the command fails instead of shrinking.
               (A LOW that lands inside an existing partition always fails.)

  BLOCKSIZE=<n>  Filesystem block size in bytes.  Defaults to 512 if
                 omitted.  Must be a power of two from 512 up to 32768
                 (512, 1024, 2048, 4096, ...).  Most setups use 512;
                 only change it if your filesystem expects larger blocks.

  Requires WRITE to commit to disk.

ADDFS TYPE=<dostype> [VERSION=<hex>] [FILE=<path>] [STACKSIZE=<n>]
  Add or replace a filesystem header (FSHD/LSEG blocks).
  FILE loads a filesystem binary (e.g. L:pfs3aio) as LSEG blocks.
  VERSION is a 32-bit hex version word (e.g. 0x00130000 for v19.0).
  Requires WRITE to commit to disk.

GROW <drive> <size[KMG]|END> [NOUNMOUNT]
  Extend an existing partition and grow its filesystem to match, in one
  step.  Works on FFS/OFS, SFS and PFS3/PFS2 partitions.
  <size>      Amount to ADD, e.g. 100M.  If it exceeds the free space
              after the partition, it is clamped to the maximum (the tool
              prints how much it actually grew).
  END (or MAX) Use all the free space up to the next partition / disk end.
  NOUNMOUNT   Grow in place without unmounting first (see below).

  The grow runs offline: the volume is unmounted, the filesystem is grown,
  the RDB is written immediately (so on-disk geometry and the filesystem
  never disagree), and FFS volumes are remounted live (no reboot).  SFS/PFS
  leave the volume inhibited and need a reboot.  Each step is printed.
  Unlike ADDPART/ADDFS, GROW commits on its own — no separate WRITE needed.
  In DRYRUN mode it prints the plan and changes nothing.

  NOUNMOUNT skips the unmount step, which the boot partition (and any volume
  with open files/locks) cannot satisfy.  The filesystem is still grown with
  the handler inhibited, then the volume is left locked and a reboot is
  required before it can be used again — do NOT write to it beforehand.

    OPEN uaehf.device 3
    GROW DH1 100M       ; add 100 MB (clamped to free space if less)
    GROW DH1 END        ; or: use all the free space after DH1
    GROW DH0 END NOUNMOUNT   ; boot partition: grow in place, then reboot
    CLOSE

ZEROPART NAME=<drive>
  Overwrite every block of the named partition with zeros.  Writes directly
  to disk using the open device — no WRITE needed, and the RDB is not
  modified.  Useful before imaging: zeros compress far better than stale
  filesystem data.
  In DRYRUN mode, prints what would be zeroed without writing anything.

    OPEN scsi.device 0
    ZEROPART NAME=DH1
    CLOSE

WRITE
  Commit all pending in-memory changes to disk.
  Without WRITE, ADDPART/ADDFS do not touch the disk.
  In DRYRUN mode, prints what would be written instead.

CLOSE
  Close the current device.  Warns if there are uncommitted changes.
  The device is also closed automatically at end of script.

REBOOT
  Ask Y/N, then reboot the machine.  FORCE auto-answers Y.
  Counts down 3 seconds before rebooting so output is visible.
  Useful at the end of a full disk-preparation script.


EXAMPLES
--------

Prepare a disk with PFS3 (single large partition)
--------------------------------------------------
Save as e.g. ram:prep_pfs3.txt, then run:
  DiskPart SCRIPT ram:prep_pfs3.txt DRYRUN   (check first)
  DiskPart SCRIPT ram:prep_pfs3.txt FORCE    (write for real)

  # -------------------------------------------------------
  # Prepare uaehf.device unit 0 as a PFS3 disk
  # Cylinder 0-1 : reserved for RDB
  # DH0          : 100 MB system partition (bootable)
  # DH1          : remainder of disk as data partition
  # -------------------------------------------------------

  OPEN uaehf.device 0
  INIT NEW

  # Load the PFS3 filesystem driver into the RDB
  ADDFS TYPE=PFS3 FILE=L:pfs3aio

  # System partition: 100 MB, bootable
  ADDPART NAME=DH0 LOW=START HIGH=+100M TYPE=PFS3 BOOTPRI=0 BOOTABLE

  # Data partition: everything that remains
  ADDPART NAME=DH1 LOW=START HIGH=END TYPE=PFS3

  WRITE
  REBOOT


Prepare a disk with PFS3 (three partitions)
--------------------------------------------
  OPEN uaehf.device 0
  INIT NEW

  ADDFS TYPE=PFS3 FILE=L:pfs3aio

  ADDPART NAME=DH0 LOW=START HIGH=+100M TYPE=PFS3 BOOTPRI=0 BOOTABLE
  ADDPART NAME=DH1 LOW=START HIGH=+500M TYPE=PFS3
  ADDPART NAME=DH2 LOW=START HIGH=END   TYPE=PFS3

  WRITE
  REBOOT
